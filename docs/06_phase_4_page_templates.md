# 06 - Phase 4: Page Templates

## Goal

Create templates that match the corrected architecture and UX.

## Template 1: Homepage `/en/`

Structure:
1. compact hero
2. browse/search prompt
3. popular content types
4. featured collections
5. seasonal collections
6. new printables
7. for teachers/parents/adults sections
8. trust strip

Hero:
- not too tall
- not huge SaaS typography
- include preview cards near top
- clear CTA: Browse Coloring Pages, Explore Printables

## Template 2: Content Type Hub

Example:
`/en/coloring-pages/`

Structure:
1. H1
2. short intro
3. hub cards
4. featured topics
5. seasonal/popular topics
6. helpful text
7. related content

## Template 3: Hub / Subcategory Page

Example:
`/en/coloring-pages/kids/`

Structure:
1. H1
2. short intro
3. topic cards
4. popular collections
5. usage tips
6. FAQ if useful

## Template 4: Topic / Collection Page

Example:
`/en/coloring-pages/kids/wizard-school-coloring-pages/`

Structure:
1. breadcrumbs
2. H1
3. short intro
4. quick facts
5. filter chips
6. printable grid early
7. individual download buttons
8. helpful usage content
9. printing tips
10. related collections
11. FAQ
12. license/trust note

No full-pack download button at MVP.

No modal/lightbox.

## Template 5: Optional Individual Printable Page

Example:
`/en/coloring-pages/kids/wizard-school-coloring-pages/magic-owl-coloring-page/`

Structure:
1. breadcrumbs
2. H1
3. large framed preview
4. Download PDF button
5. Print button optional
6. short description
7. belongs-to collection link
8. related printables
9. license note

Default:
- noindex unless unique
- canonical to topic/collection page when thin

## Template 6: Legal/Trust Pages

Required:
- About
- Contact
- Privacy Policy
- Terms
- Licensing
- DMCA/Takedown
- How to Print
- AI-Assisted Content Policy

No backend contact form in MVP.

## Phase 4 Done Criteria

- all MVP templates exist
- pages feel like a catalogue
- printables appear early
- SEO structure is clean
- no forbidden MVP features
