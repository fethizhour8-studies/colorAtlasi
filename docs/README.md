# coloringsheetsV3 Orchestrator Docs

These files are the updated master instructions for the printable/coloring website.

They replace the previous version.

Major corrections included:
- Architecture is now: content type → hub/subcategory → topic/collection → optional individual printable.
- A topic page can also be the collection page.
- Separate pack/subcollection pages are optional, only for large topics.
- No full-pack download button at MVP.
- No lightboxes/modals at MVP.
- Collection/topic pages are grid-first scrollable catalogue pages.
- Individual PDF downloads appear under each printable card.
- Grid structure is prepared for future in-feed ads, but no live ads are added in MVP.
- No multilingual UI in MVP; keep `/en/` URL structure only.
- Python is now the preferred asset pipeline.
- The design direction is warmer, more colorful, more printable-focused, and less SaaS-like.
