# Final Launch Checklist

Use this checklist before uploading the static build to production.

## 1. Domain and environment

- Pick the final domain before launch.
- Set `SITE_URL` to the exact production origin, with no trailing slash.
- Set `PUBLIC_CONTACT_EMAIL` to the public support/contact inbox.
- Keep `.env` out of Git.

Recommended domain direction:

1. EasyColoringPages.com
2. PrintColoring.com
3. SimpleColoringPages.com
4. ColoringPagePrints.com
5. ColoringShelf.com

Use a coloring-focused domain if the first launch is mainly coloring pages. Use a broader printable-focused domain only if worksheets, planners, and calendars will become equally important soon.

## 2. Build commands

```bash
npm install
npm run build
```

For production URL testing on Windows PowerShell:

```powershell
$env:SITE_URL="https://your-domain.com"
npm run build
```

For macOS/Linux:

```bash
export SITE_URL="https://your-domain.com"
npm run build
```

## 3. Files that should exist in `dist/`

- `index.html` or route folders generated by Astro
- `sitemap.xml`
- `robots.txt`
- `llms.txt`
- `_headers`
- `.htaccess`
- static asset folders

## 4. SEO and GEO checks

Check several pages in browser view-source:

- canonical URL is absolute and uses the production domain
- meta description exists
- robots value is correct
- Open Graph tags exist
- Twitter tags exist
- JSON-LD scripts exist
- sitemap includes indexable pages
- planned/noindex topics do not become priority SEO targets until they have real printables

Recommended pages to inspect:

- `/en/`
- `/en/coloring-pages/`
- `/en/coloring-pages/animals/`
- `/en/coloring-pages/animals/farm-animals/`
- `/en/coloring-pages/kids/wizard-school-coloring-pages/`
- `/en/coloring-pages/kids/wizard-school-coloring-pages/magic-owl-coloring-page/`
- `/sitemap.xml`
- `/robots.txt`
- `/llms.txt`

## 5. Analytics and downloads

Optional Plausible setup:

```bash
PUBLIC_PLAUSIBLE_DOMAIN=your-domain.com
PUBLIC_PLAUSIBLE_SRC=https://plausible.io/js/script.file-downloads.outbound-links.js
```

Track:

- page views by collection
- PDF downloads
- top referrers
- outbound clicks if enabled by analytics script
- server 404s from hosting logs

Do not collect personal data unless privacy/cookie policies are updated.

## 6. Content quality checks

Before indexing a new topic, make sure it has:

- real approved PDF assets
- at least one strong preview image
- original non-infringing concept
- no protected characters, logos, celebrities, brands, or franchise visual designs
- helpful intro
- quick facts
- printing guidance
- related internal links
- clean page title and meta description

Keep planned topics `indexable: false` until there are enough real assets.

## 7. Upload/deploy safety

Upload only the static build output.

Do not upload:

- `.git/`
- `node_modules/`
- source files unless your host builds from Git
- `.env`
- raw API keys
- raw generation manifests
- screenshots folder

If using cPanel/Apache hosting, confirm `.htaccess` is included. If using Netlify/Cloudflare Pages, confirm `_headers` is included.

## 8. Post-launch tasks

- Verify HTTPS.
- Submit sitemap in Google Search Console.
- Check mobile usability.
- Check page speed after real images are added.
- Check social share preview for a topic page.
- Monitor 404 logs daily for the first week.
- Expand only the topics that get traffic or downloads.

## 9. Next growth priorities

1. Add 4 to 8 real PDFs per high-priority topic.
2. Turn strong planned topics from `indexable: false` to `true` only after content is real.
3. Add Pagefind search after the catalogue has enough pages.
4. Add language routing later, starting with one language and only top pages.
5. Add ad placeholders only after content, UX, and analytics are stable.
